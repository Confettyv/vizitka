# Handoff: Сайт-визитка fullstack-разработчика (Alim)

## Overview
Персональный сайт-визитка fullstack-разработчика для привлечения доверия заказчиков с фриланс-площадок. Состоит из лендинга и двух страниц кейсов. Главные смыслы: скорость запуска продуктов, честная подача (реальные проекты вместо выдуманной статистики), двуязычность (RU/EN). Услуги: сайты, AI рекламные видео, Telegram-боты, Discord-боты.

## About the Design Files
Файлы в этом пакете (`*.dc.html`) — это **дизайн-референсы, созданные в HTML на кастомном прототип-рантайме**. Это НЕ production-код для прямого копирования. Задача — **пересоздать эти дизайны в целевом окружении** (рекомендуется React + Vite или Next.js, т.к. сайт статичный), используя обычные компоненты и инлайн/CSS-стили. Если репозитория ещё нет — создать новый на React/Next, перенести разметку и стили из этих файлов 1:1 по визуалу.

Динамика в прототипах сделана на маленьком классе-логике (`class Component`), но в проде это превращается в обычные React-компоненты с `useState`/`useEffect`. Игнорируй рантайм-обёртку (`support.js`, теги `<x-dc>`, `<helmet>`, `<x-import>`) — бери из файлов **разметку, инлайн-стили, копирайт и логику поведения**.

## Fidelity
**High-fidelity (hifi).** Финальные цвета, типографика, отступы и интерактив. Воссоздавать пиксель-в-пиксель. Все точные значения — ниже в Design Tokens.

## Interactions & Behavior
- **RU/EN-переключатель.** Каждый переводимый текст-элемент в прототипе помечен атрибутом `data-en="English text"`, оригинал (RU) сохраняется в `data-ru` при монтировании. По клику RU/EN меняется `textContent` всех `[data-en]` и стили двух кнопок-переключателей. **В проде:** замени на нормальный i18n — словарь строк `{ ru, en }` и состояние `lang` (например `react-i18next` или простой Context). Все RU/EN-пары копий собраны в самих файлах в атрибутах `data-en` / тексте элемента.
- **Sticky-хедер с авто-скрытием.** Скролл-листенер: при скролле вниз (за 80px) хедер уезжает вверх (`translateY(-(height+4)px)`), при скролле вверх — возвращается (`translateY(0)`). `transition: transform 0.35s cubic-bezier(0.4,0,0.2,1)`.
- **Hover карточек услуг**: border темнеет, тень, подъём на 3px.
- **Появление hero** — `@keyframes slrise` (opacity 0→1, translateY 14px→0), длительность 0.5–0.6s, каскадные задержки.
- **Пульс статус-точки** — `@keyframes slpulse` (scale 1→1.35, opacity 1→0.55), 2s infinite.
- **Якорная навигация** — `html { scroll-behavior: smooth }`, ссылки на `#services #work #pricing #top`.
- **Email-антибот**: в прототипе адрес собирается из `data-user`+`data-domain` в JS (артефакт среды). **В проде это не нужно** — пиши `miroxby@gmail.com` обычной `mailto:`-ссылкой (или прикрути форму с отправкой на backend).

## State Management
Минимум: одно состояние `lang: 'ru' | 'en'` (i18n). Для хедера — отслеживание направления скролла (`lastScrollY` + listener). Без фетча данных — сайт статичный. Если добавляешь контактную форму — состояние полей + сабмит на backend/сервис.
